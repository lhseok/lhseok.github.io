애플 개발자라고 하면 보통 아이폰이나 맥 앱을 만드는 사람을 떠올린다. 워치 앱까지는 그래도 좀 보이는데, 의외로 '애플 TV(tvOS)' 앱을 만든다는 사람은 가물에 콩 나듯 귀하다. 나 역시 그랬다. 손안의 화면이나 책상 위의 모니터는 익숙하지만, 거실 한복판에 놓인 거대한 스크린을 위한 앱을 만드는 일은 왠지 모를 거리감이 느껴졌다.

하지만 최근 애플 TV 모델들이 고사양화되고 홈 허브로서의 역할이 커지면서 tvOS 앱 개발에 대한 호기심이 생겼다. 직접 겪어본 tvOS 개발은 iOS와 비슷하면서도 완전히 다른 규칙이 지배하는 세계였다.

![Apple TV on a wooden stand](https://images.unsplash.com/photo-1593359677879-a4bb92f829d1?q=80&w=1280&auto=format&fit=crop!!ar=2.2:1)

### 1. 우리가 개발해야 할 애플 TV 라인업

tvOS 앱을 개발하기 전에 우리가 타겟팅하는 하드웨어가 무엇인지 정확히 알 필요가 있다. 현재 시장에서 유효한 애플 TV 모델들을 정리해 보았다.

| 모델 | 출시년도 | 프로세서 | 특징 |
| :--- | :--- | :--- | :--- |
| **Apple TV HD** | 2015 | A8 | 1080p 해상도, tvOS의 시작 |
| **Apple TV 4K (1세대)** | 2017 | A10X Fusion | 4K HDR 지원 시작 |
| **Apple TV 4K (2세대)** | 2021 | A12 Bionic | 새로운 시리 리모트, 고프레임 HDR |
| **Apple TV 4K (3세대)** | 2022 | A15 Bionic | HDR10+, 팬리스 설계, USB-C 리모트 |

사실상 최신 앱을 쾌적하게 돌리려면 **`Apple TV 4K`** 라인업을 기준으로 생각하는 것이 편하다. 특히 3세대는 성능이 워낙 좋아서 웬만한 고사양 게임이나 복잡한 UI도 부드럽게 돌아간다.

### 2. tvOS 개발, iOS와 무엇이 다른가?

iOS 개발자라면 tvOS 프로젝트를 처음 열었을 때 "어, 똑같네?"라고 생각하기 쉽다. 하지만 그 생각은 코드 몇 줄만 짜보면 금방 깨진다. tvOS는 이른바 '**10피트 UI(10-foot UI)**' 환경이다. 사용자가 화면에서 약 3미터 정도 떨어져 앉아 있다는 뜻이다.

#### 터치가 없다, 포커스만 있을 뿐
가장 큰 차이는 입력 방식이다. 아이폰은 손가락으로 직접 터치하지만, 애플 TV는 시리 리모트(Siri Remote)의 터치패드를 스와이프하거나 클릭해서 조작한다. 여기서 '**포커스 엔진(Focus Engine)**'이라는 개념이 등장한다. 내가 어떤 버튼을 누를지 선택하는 과정이 화면에 시각적으로(버튼이 커지거나 그림자가 생기는 등) 드러나야 한다.

#### WebKit이 없다
놀랍게도 tvOS에는 `WKWebView` 가 없다. 즉, 웹 뷰를 띄워서 대충 하이브리드 앱으로 때우는 방식이 불가능하다. 모든 UI는 네이티브로 짜야 하거나, 아니면 애플이 제공하는 TVML(TV Markup Language)을 써야 한다. 개발자 입장에서는 무조건 네이티브 앱을 강요받는 셈인데, 오히려 이 점이 tvOS 앱의 퀄리티를 상향 평준화시킨다.

#### 로컬 저장소의 제약
애플 TV는 개인용 기기라기보다는 공용 기기에 가깝다. 그래서 앱이 사용할 수 있는 로컬 저장 공간이 매우 제한적이고, 시스템이 언제든 데이터를 지울 수 있다. 중요한 데이터는 무조건 **`CloudKit`** 이나 외부 서버 API를 통해 동기화해야 한다.

### 3. SwiftUI로 만드는 tvOS Hello World

이제 실제로 코드를 짜보자. Xcode에서 새로운 프로젝트를 만들 때 플랫폼을 tvOS로 선택하면 된다. SwiftUI를 사용하면 tvOS의 포커스 엔진을 아주 쉽게 다룰 수 있다.

```swift
import SwiftUI

struct ContentView: View {
    @State private var message = "Hello, Apple TV!"

    var body: some View {
        VStack(spacing: 50) {
            Text(message)
                .font(.largeTitle)
                .padding()

            HStack {
                Button(action: {
                    message = "안녕하세요!"
                }) {
                    Text("한국어")
                }
                
                Button(action: {
                    message = "Hello World!"
                }) {
                    Text("English")
                }
            }
        }
    }
}
```

이 간단한 코드를 애플 TV 시뮬레이터에서 실행해 보면, 리모컨 방향키(스와이프)에 따라 버튼이 커지고 작아지는 것을 볼 수 있다. SwiftUI의 `Button` 은 기본적으로 tvOS의 포커스 스타일을 지원하기 때문에 별다른 설정 없이도 'TV 앱 같은 느낌'이 난다.

### 4. 개발 시 유의사항과 팁

- **안전 영역(Safe Area)**: TV는 화면 끝부분이 베젤에 가려지거나 왜곡되는 현상(Overscan)이 있을 수 있다. 그래서 중요한 UI 요소는 화면 가장자리에서 충분히 띄워야 한다.
- **다크 모드와 라이트 모드**: 사용자는 거실 조명 상태에 따라 테마를 바꾼다. 두 환경 모두에서 가독성이 확보되는지 반드시 체크해야 한다.
- **리모컨 제스처**: 단순히 클릭뿐만 아니라 리모컨의 재생/일시정지 버튼, 메뉴 버튼에 대한 대응도 고민해야 한다. 특히 메뉴 버튼은 '뒤로 가기' 역할을 수행해야 하므로 내비게이션 구조 설계가 중요하다.

### 5. 빌드와 테스트

애플 TV 실기기가 있다면 Xcode와 동일한 네트워크(Wi-Fi)에 연결하여 무선으로 빌드할 수 있다. 예전에는 USB-C 포트가 본체에 있어서 유선 연결이 가능했지만, 최신 4K 모델은 포트가 없어서 무선 연결이 필수다. 시뮬레이터로도 충분히 개발이 가능하지만, 포커스 엔진의 실제 '손맛'을 느끼려면 실기기 테스트를 강력히 권장한다.

낯선 환경은 언제나 두려움과 설렘을 동시에 준다. tvOS는 비록 주류 플랫폼은 아니지만, 거실의 큰 화면을 점유한다는 매력이 분명히 있다. 내가 만든 앱이 거실의 TV에서 구동되는 모습은 아이폰 화면과는 또 다른 감동을 준다.

> "거실이라는 공간을 점유하는 앱을 만든다는 것, 그것은 사용자 삶의 가장 편안한 시간에 스며드는 일이다."
