개발자에게 코드 리뷰와 디버깅은 피할 수 없는 숙명과도 같다. 하지만 때로는 쏟아지는 코드 사이에서 길을 잃기도 하고, 단순 반복적인 리팩터링 작업에 지쳐 정작 중요한 로직 설계에 소홀해지기도 한다. 나 역시 GitHub Copilot이나 Cursor 같은 도구들을 전전하며 최적의 'AI 파트너'를 찾아 헤맸다. 그러다 만난 **Claude Code**는 기존의 도구들과는 확연히 다른 충격을 선사했다.

![AI text written in sand](https://images.unsplash.com/photo-1740560516633-4d5a7d98c8f4?q=80&w=1280&auto=format&fit=crop)

### 1. 왜 채팅이 아니라 CLI인가?

처음 Claude Code가 CLI(Command Line Interface) 형태로 나왔을 때, 나는 의구심이 들었다. "요즘 세상에 GUI도 아니고 굳이 터미널에서?"라는 생각이었다. 하지만 단 하루만 써보고 그 생각이 완전히 바뀌었다.

터미널 기반의 AI는 **맥락(Context)의 접근성**이 다르다. 웹 채팅창에 코드를 복사해서 붙여넣을 필요가 없다. Claude Code는 내 프로젝트의 파일 구조를 꿰뚫어 보고, 필요한 파일을 스스로 읽으며, 수정을 제안하고, 심지어는 테스트 명령어를 실행해 그 결과까지 확인한다. 개발자가 IDE와 브라우저를 오가는 '컨텍스트 스위칭' 비용을 혁신적으로 줄여주는 것이다. "이 함수랑 연결된 모든 파일을 다크모드 대응해줘"라고 한마디만 하면, Claude는 수십 개의 파일을 넘나들며 작업을 수행한다.

### 2. CLAUDE.md: 프로젝트의 영혼을 담는 그릇

Claude Code를 쓰면서 가장 감탄한 기능은 바로 `CLAUDE.md` 파일이다. 이 파일은 단순한 문서가 아니다. AI에게 내 프로젝트의 '철학'과 '규칙'을 주입하는 온보딩 가이드라인이다.

나는 `CLAUDE.md`에 기술 스택뿐만 아니라, 내가 선호하는 네이밍 규칙, 아키텍처 패턴, 심지어는 "이 라이브러리는 절대 쓰지 말 것" 같은 금기 사항까지 상세히 적어두었다. 그랬더니 놀라운 일이 벌어졌다. 이전까지는 AI가 짜준 코드를 제 스타일에 맞게 일일이 수정해야 했지만, 이제는 Claude가 처음부터 내가 짠 것 같은 코드를 내놓기 시작한 것이다. 

예를 들어, "새로운 뷰 컴포넌트를 만들어줘"라고 요청하면, `CLAUDE.md`에 명시된 대로 SwiftUI의 `@Observable` 매크로를 사용하고, 폴더 구조까지 정확히 지켜서 파일을 생성해준다. 이것은 단순한 생산성 향상을 넘어, AI와 나 사이의 '**팀워크**'가 형성되는 경험이었다.

### 3. "Aha!" 모먼트: 3시간짜리 버그를 3분 만에 잡다

얼마 전, 비동기 데이터 처리 과정에서 발생하는 간헐적인 레이스 컨디션(Race Condition) 때문에 골머리를 앓고 있었다. 로그를 찍어봐도 원인이 명확하지 않아 3시간째 헤매고 있었다. 지푸라기라도 잡는 심정으로 Claude Code에게 상황을 설명했다.

> "지금 데이터 업데이트가 가끔 누락되는데, 뷰모델의 Combine 퍼블리셔랑 로컬 DB 저장 로직을 분석해서 병목이나 경합이 발생하는 지점을 찾아줘."

Claude는 순식간에 관련 파일 5개를 읽어 내려갔고, 내가 놓치고 있었던 `receive(on: RunLoop.main)` 위치의 부적절함과 그로 인해 발생하는 미세한 타이밍 차이를 지적해냈다. 수정 제안과 함께 말이다. 코드를 적용하자마자 버그는 감쪽같이 사라졌다. 그때 깨달았다. Claude Code는 단순한 보조 도구가 아니라, 나보다 훨씬 냉철하게 전체 구조를 파악하는 '**시니어 파트너**'라는 사실을 말이다.

### 4. 나가는 글: 이제는 '코딩'이 아니라 '오케스트레이션'이다

Claude Code를 사용하면서 저의 역할은 점점 변하고 있다. 직접 타이핑을 하는 'Worker'에서, AI에게 명확한 목표를 제시하고 결과를 검증하는 'Orchestrator'로 말이다. 

물론 AI가 모든 것을 완벽하게 해결해주지는 않는다. 가끔 엉뚱한 파일을 건드리기도 하고, 복잡한 로직에서 갈팡질팡하기도 한다. 하지만 CLI 환경에서 AI와 실시간으로 소통하며 문제를 해결해 나가는 과정은 개발의 즐거움을 새로운 차원으로 끌어올려 주었다. 

> "도구는 진화하고, 개발자의 정의는 다시 쓰이고 있다. 터미널은 이제 단순히 명령어를 입력하는 곳이 아니라, AI와 함께 창조하는 무대다."
