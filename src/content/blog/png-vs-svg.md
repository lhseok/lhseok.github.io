웹 디자인과 프론트엔드 개발을 하다 보면 항상 마주치는 고민이 있다. "이 로고는 PNG로 저장할까, 아니면 SVG로 저장할까?" 언뜻 보기엔 비슷해 보여도, 이 두 형식은 태생부터가 완전히 다르다. 하나는 픽셀(Pixel)이라는 작은 점들의 집합이고, 다른 하나는 수학적인 수식(Path)의 집합이다. 오늘은 내가 실무에서 겪은 에피소드와 함께, 이 두 이미지 형식의 명암을 확실히 짚어보려 한다.

![Raster vs Vector Comparison](https://images.unsplash.com/photo-1558655146-d09347e92766?q=80&w=1000&auto=format&fit=crop)

### 1. 실전 경험: 대형 화면에서 마주한 '픽셀의 공포'

예전에 중요한 프로젝트 발표를 위해 디자인 시안을 준비했을 때의 일이다. 내 노트북의 13인치 화면에서는 선명하고 완벽해 보이던 로고가, 회의실의 대형 4K 모니터에 띄워지는 순간 형체를 알아보기 힘들 정도로 뭉개져 보였다. 로고의 테두리에는 지저분한 '계단 현상(Aliasing)'이 일어났고, 팀원들의 눈길이 그곳에 머무는 것만 같아 얼굴이 화끈거렸다.

그때 사용했던 로고는 고해상도라고 믿었던 1024x1024 픽셀의 PNG 파일이었다. 하지만 '래스터(Raster)' 방식인 PNG는 정해진 픽셀 수 이상으로 확대하면 물리적인 한계에 부딪힌다. 반면, 함께 배치했던 텍스트와 벡터 기반의 아이콘들은 어떤 크기에서도 칼날처럼 날카로운 선명함을 유지하고 있었다. 그날 이후, 나는 로고나 아이콘처럼 '**형태의 본질**'이 중요한 요소에는 무조건 SVG를 쓰기로 마음먹었다.

### 2. SVG는 단순한 이미지가 아니라 '코드'다

SVG의 진정한 위력은 그것이 이미지가 아니라 '**텍스트로 작성된 설계도**'라는 점에 있다. 이는 프론트엔드 개발자에게 엄청난 자유도를 부여한다. 이미지 파일을 다시 뽑지 않고도 CSS나 JavaScript로 내부 요소를 마음껏 주무를 수 있기 때문이다.

```css
/* CSS로 마우스 호버 시 SVG 내부 경로 색상 변경 */
.icon-svg:hover path {
  fill: #ff4757;
  transition: fill 0.3s ease;
}
```

예를 들어, 다크 모드에 따라 아이콘 색상을 바꿔야 할 때 PNG라면 두 장의 이미지가 필요하겠지만, SVG는 CSS 한 줄이면 충분하다. 또한 JavaScript를 이용해 선이 그려지는 듯한 애니메이션을 구현할 수도 있다.

```javascript
/* JavaScript로 SVG 경로 애니메이션 제어 */
const path = document.querySelector('.logo-path');
if (path) {
  const length = path.getTotalLength(); // 전체 경로 길이 계산
  
  // 드로잉 애니메이션 설정
  path.style.strokeDasharray = length;
  path.style.strokeDashoffset = length;
}
```

### 3. 복잡성의 함정: 언제 PNG로 돌아가야 하는가?

하지만 SVG가 항상 만능은 아니다. 치명적인 함정이 하나 있는데, 바로 '**데이터의 양**'이다. 만약 수만 개의 점과 복잡한 질감이 들어간 풍경 사진을 SVG로 변환하면 어떻게 될까? 브라우저는 그 수만 개의 수학적 경로를 계산하느라 비명을 지르게 될 것이다. 

이런 경우에는 래스터 방식인 PNG(또는 최신 포맷인 WebP)가 훨씬 효율적이다. PNG는 픽셀의 색상 정보를 그대로 담고 있기 때문에, 아무리 복잡한 이미지라도 브라우저는 그저 '픽셀을 뿌려주기만' 하면 된다. 복잡한 연산 과정이 거의 없다. 따라서 고화질 사진이나 세밀한 그라데이션이 들어간 그래픽은 PNG로 처리하는 것이 성능과 퀄리티 면에서 유리하다.

### 4. 실무 선택 가이드 요약

상황에 따라 어떤 형식을 골라야 할지 헷갈린다면 아래의 기준을 따라보자.

- **SVG를 선택해야 할 때**:
  - 로고, 아이콘, 타이포그래피, 단순한 기하학적 도형
  - 인터랙티브한 애니메이션이 필요한 경우
  - 다양한 해상도의 기기에 대응해야 할 때 (반응형 웹)
- **PNG를 선택해야 할 때**:
  - 인물, 풍경 등 색상이 다채로운 실제 사진
  - 투명 배경이 필요하지만 이미지가 너무 복잡해 SVG로 감당이 안 될 때
  - 특정 필터 효과나 텍스처 질감이 중요한 디자인

### 나가는 글: 도구의 이해가 퀄리티를 만듭니다

결국 이미지 형식의 선택은 '**품질과 성능 사이의 줄타기**'다. 선명함을 위해 용량을 포기할 것인가, 아니면 성능을 위해 디테일을 타협할 것인가. 

> "도구의 본질을 이해하는 것이 곧 퀄리티로 이어진다. 뭉개진 픽셀 속에 숨겨진 디테일을 찾아 SVG로 교체하는 순간, 여러분의 디자인은 한 단계 더 진화할 것이다."
