Gemini CLI, Claude Code 등 최근 터미널에서 AI 바이브 코딩을 자주 하다 보니 GUI에 익숙하던 내가, 터미널을 수도 없이 띄우기 시작했다. 예전에는 마우스만으로 딸깍하며 실행하는 게 당연했는데, 터미널 창 안에서 모든 걸 해결하는 맛을 들이고 나니 이제는 마우스에 손이 가는 것 자체가 흐름을 깨는 일처럼 느껴진다.

하지만 터미널과 친해지는 과정이 그리 순탄치만은 않았다. 가장 먼저 마주친 장벽은 매번 깊숙이 숨어 있는 내 작업 폴더(GitHub 디렉토리 등)로 찾아 들어가는 일이었다. 더구나 프로젝트가 여러 개라면 더더욱.

![Terminal window on a laptop screen](https://images.unsplash.com/photo-1629654297299-c8506221ca97?q=80&w=1280&auto=format&fit=cro!!ar=3:1)

### 1. 무식하면 몸이 고생하는 cd 명령어의 굴레

처음에는 무식하게 **`cd`** 명령어를 연타했다.

```bash
cd Documents
cd GitHub
cd TestProject
cd flutter_app
```

매번 프로젝트를 전환할 때마다 이 긴 경로를 타이핑하거나, 이전 명령어를 위아래 방향키로 찾아 헤매는 건 여간 귀찮은 일이 아니었다. 탭(Tab) 키를 활용한 자동 완성이 있다고 해도 한계가 있었다. 아이폰에서의 단축어나 윈도우의 숏컷 같은 기능이 간절했다.

역시나 있었다. 리눅스나 맥 같은 유닉스 계열 OS에는 **`alias`** 라는 편리한 기능이 존재한다.

### 2. alias: 터미널판 '바로가기 아이콘'

**`alias`** 는 쉽게 말해 터미널판 '바로가기 아이콘' 또는 '별명'이다. 바탕화면에 있는 복잡한 경로의 프로그램도 바로가기 아이콘 하나만 더블클릭하면 실행되듯, 터미널에서도 길고 복잡한 명령어에 나만의 짧은 별명을 붙여 실행할 수 있게 해준다.

예를 들어 내 작업 폴더들로 한 번에 점프할 수 있는 별명을 아래처럼 지어줄 수 있다.

```bash
alias gotoft="cd ~/Documents/GitHub/TestProject/flutter_app"
alias gotoleemily="cd ~/Documents/GitHub/LEEMILY"
alias gotoapp="cd ~/Documents/GitHub/TestApp"
alias gotorepo="cd ~/Documents/GitHub/TestRepo"
```

이렇게 터미널에 한 줄씩 입력해두면, 앞으로 **`cd Documents/GitHub/...`** 를 길게 칠 필요 없이 그냥 **`gotoleemily`** 라고만 입력해도 해당 폴더로 즉시 워프한다. 타자 수가 획기적으로 줄어들고 코딩 텐션이 유지된다.

### 3. 터미널을 꺼도 유지되게 만드는 .zshrc 파일

방금 터미널에 직접 **`alias`** 명령어를 입력해 감탄하고 있었다면, 터미널 창을 껐다가 다시 켜보자. 그리고 방금 등록한 명령어를 치면 보기 좋게 배신을 당한다. 터미널에 직접 입력한 alias는 현재 열려 있는 '터미널 세션'이 살아있을 때만 유지되는 일회성 세팅이기 때문이다. 

컴퓨터를 껐다 켜도, 터미널을 백 번 껐다 켜도 내 소중한 단축어들이 그대로 살아있게 하려면 터미널의 '기본 설정 파일'에 이 내용을 박아두어야 한다. 맥북을 쓰고 있다면(특히 카탈리나 OS 이후 버전) 기본 쉘로 Zsh를 사용한다. 이 Zsh 쉘이 켜질 때마다 컴퓨터가 가장 먼저 읽어 들이는 텍스트 파일이 바로 **`~/.zshrc`** 다.

이름을 뜯어보면 의미가 명확하다.

- **.** (마침표): 중요한 설정 파일이니 평소에는 안 보이게 숨겨두겠다는 뜻(숨김 파일).
- **zsh**: Z 쉘 전용 파일이라는 뜻.
- **rc**: Run Command의 약자로, 프로그램이 시작될 때 자동으로 실행할 명령들을 모아둔 파일에 붙는 유구한 전통의 접미사다.

쉽게 말해 **`.zshrc`** 파일은, 터미널에게 구동시마다 이 파일 안에 적힌 명령어들 위에서부터 차례대로 실행하고 시작하라고 지정해 두는, 윈도우에서의 시작 프로그램과 같다. 설정하는 방법은 간단하다. 홈 디렉토리에 있는 **`.zshrc`** 파일을 에디터로 열고 맨 밑에 코드를 추가해주면 된다.

```bash
nano ~/.zshrc
```

파일이 열리면 맨 아래에 아까 만들었던 alias 모음을 추가하고 저장(**`Ctrl + O`** , **`Enter`** , **`Ctrl + X`** )하면 끝이다.

### 4. 설정의 완성, source 명령어

파일도 저장 후 터미널에 방금 입력한 단축어를 입력하면 이런 에러가 뜰거다.

```bash
leemily@leemilyui-MacBookPro GitHub % gotoleemily
zsh: command not found: gotoleemily
```

분명히 파일에 똑바로 저장했는데 단축어를 찾을 수 없다고 나온다. 이유는 단순하다. 파일은 수정되었지만, 지금 내가 보고 있는 이 터미널 창은 수정되기 전(터미널이 켜지던 시점)의 과거 **`.zshrc`** 파일을 읽은 상태이기 때문이다. 새 설정이 적용되려면 터미널을 완전히 종료했다가 새로 열거나, 현재 열려 있는 창에 "파일 새로고침"을 해줘야 한다. 이때 사용하는 명령어가 바로 **`source`** 다.

```bash
leemily@leemilyui-MacBookPro GitHub % source ~/.zshrc
leemily@leemilyui-MacBookPro GitHub % gotoleemily  
leemily@leemilyui-MacBookPro LEEMILY % 
```

**`source ~/.zshrc`** 를 치자마자 먹통이던 **`gotoleemily`** 가 기가 막히게 잘 작동하며 폴더를 이동한다. **`source`** 명령어는 브라우저의 '새로고침' 같은 역할을 한다. 터미널을 끄지 않고도 방금 내가 수정한 파일을 지금 즉시 다시 읽어서 현재 창에 반영하라는 강제 동기화 명령이다. 참고로 **`source`** 대신 점 한 개(`.`)를 찍어서 **`. ~/.zshrc`** 라고 입력해도 동일하게 동작한다.

### 마치며

매번 마우스로 파인더를 실행하고 폴더를 찾아 앱을 켜던 시절에서, 이제는 터미널을 열자마자 단 몇 글자만 툭 치고 바로 Claude Code나 Gemini 올려 코딩을 시작한다. 별것 아닌 세팅 같지만, 개발 환경에서 마우스로 손이 가는 횟수를 줄이고 타이핑 효율을 높이는 것이 몰입도를 유지하는 데 얼마나 큰 도움이 되는지 최근에야 다시 새삼 깨닫는다. 혹시 나처럼 터미널에서 긴 경로를 무식하게 치고 있었다면, 지금 당장 **`.zshrc`** 를 열어 나만의 지름길을 만들어두자. 생산성이 배로 올라간다. 불과 몇 초 차이일지라도 그 몇초가 매일 매달 쌓이다보면 무시할 수 없는 시간이 된다.

> "터미널은 도구가 아니라 환경이다. 환경이 편안해질 때 비로소 코딩에 온전히 집중할 수 있다."
